# Quickstart

Step-by-step local dev setup. Assumes Windows 11; macOS/Linux work the same.

## Prereqs

| Tool | Version | Install |
|---|---|---|
| Node.js | 20.x | https://nodejs.org/ |
| pnpm | 9.x | `npm install -g pnpm` |
| Godot | 4.3+ | https://godotengine.org/download |

No Rust / Anchor / Solana CLI needed — the server uses plain
`@solana/web3.js`, and keypairs are generated by `pnpm run keys:gen`.

Windows: install Godot to `C:\Tools\Godot\` and add to PATH.

## 1. Clone + install

```powershell
git clone <your-fork>/kartchain.git
cd kartchain
pnpm install
```

## 2. Generate the custodial keypairs

```powershell
pnpm run keys:gen
```

This writes four JSON keypair files into `server/`:

| File | Wallet |
|---|---|
| `server/treasury.json` | Treasury (fee accumulator) |
| `server/escrow.json` | Race escrow (entry fees + payouts) |
| `server/p2e-mint.json` | P2E SPL token mint authority |
| `server/nft-auth.json` | Kart NFT mint authority (optional) |

> **Never commit these files.** They are already in `.gitignore`.

## 3. Fund escrow + treasury on devnet

The escrow wallet needs SOL to pay tx fees + a starting buffer. Treasury
also needs a tiny balance to pay ATA-creation rent when minting P2E rewards.

Two options:

**Option A — use the Solana CLI (if installed):**
```powershell
solana config set --url devnet
solana airdrop 2 (solana-keygen pubkey server\treasury.json)
solana airdrop 2 (solana-keygen pubkey server\escrow.json)
```

**Option B — paste pubkeys into a faucet:**
- https://faucet.solana.com (devnet)
- Get pubkeys with: `node -e "console.log(require('@solana/web3.js').Keypair.fromSecretKey(Uint8Array.from(require('./server/treasury.json'))).publicKey.toBase58())"`

## 4. Create the P2E SPL token

The P2E reward token uses the `p2eMint` keypair as its mint authority.

**Option A — Solana CLI:**
```powershell
spl-token create-token --decimals 9 --mint-authority (solana-keygen pubkey server\p2e-mint.json) --url devnet
```
Paste the printed mint address into `.env` as `P2E_TOKEN_MINT=...`.

**Option B — skip for now:** leave `P2E_TOKEN_MINT` empty in `.env`.
P2E minting becomes a no-op (race settlement still works for SOL prize pool).

## 5. Configure env files

```powershell
Copy-Item .env.example .env
Copy-Item .env.example server\.env
Copy-Item .env.example web\.env.local
```

Edit each to set:
- `SOLANA_RPC_URL` — devnet default works, or use a Helius/Triton URL
- `P2E_TOKEN_MINT` — from step 4 (or leave blank)
- `ADMIN_TOKEN` — random string for `/api/admin/mint-kart` auth

## 6. Start the Colyseus server

```powershell
pnpm dev:server
# Listening on ws://localhost:2567
```

You should see lines like:
```
[wallets] treasury=8x...
[wallets] escrow  =Hg...
[wallets] p2eMint =Lf...
[kartchain-server] listening on http://localhost:2567
```

## 7. Build the Godot client

```powershell
pnpm build:client
```

This produces the WASM bundle inside `web/public/game/`.

> **Note:** Godot's Web export requires `Cross-Origin-Opener-Policy: same-origin`
> and `Cross-Origin-Embedder-Policy: require-corp` headers. Next.js is already
> configured in `web/next.config.mjs` to send these for `/game/*`.

## 8. Run the web host

```powershell
pnpm dev:web
# http://localhost:3000
```

## 9. Race!

1. Visit http://localhost:3000
2. Click "Connect Wallet" (Phantom on devnet)
3. Go to /race → pick a lobby → click Join
   - Free lobby: joins instantly
   - Paid lobby: wallet pops up to sign a `SystemProgram.transfer` to escrow
4. Drive: WASD or arrow keys, Space for items

## Hot reload

- **Server:** `pnpm dev:server` uses `tsx watch` — saves restart automatically.
- **Web:** `pnpm dev:web` is Next.js dev mode — Fast Refresh on save.
- **Godot:** rebuild client (step 7) after edits. We don't yet hot-reload the
  WASM bundle in the web host.

## Troubleshooting

**Server says `Could not read escrow keypair at ...`**
You skipped `pnpm run keys:gen`, or you started the server from a directory
other than `server/`. Run keys:gen, then `cd server && pnpm dev`.

**Server logs say P2E mint failed: `P2E_TOKEN_MINT env var is not set`**
You skipped step 4. Either create the SPL token or unset the variable to
silence the warning — P2E minting is opt-in.

**Wallet adapter doesn't show Phantom**
Phantom only injects on https or localhost. Confirm you're at
`http://localhost:3000`, not `127.0.0.1:3000`.

**`Cross-Origin-Embedder-Policy` errors in console**
Check `web/next.config.mjs` — headers must be served from the route hosting
the Godot canvas.

**Godot exports are huge (40 MB+)**
Strip unused features in `client/export_presets.cfg` — disable webxr,
multiplayer (if not using godot net), 3D physics jolt, etc.
